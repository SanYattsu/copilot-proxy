# Proxy Server and 3x-UI Panel Deployment with Ansible

This Ansible playbook automates the deployment of a proxy server using **3proxy** and a **3x-UI** (Xray) panel, both running in Docker Compose.

## Prerequisites

1. Ansible installed on your local machine.
2. SSH access to the target VPS.
3. Linux distribution - **Debian/Ubuntu** or you need to manually update Docker installation vars.
4. Required Ansible collections:

   ```bash
   ansible-galaxy collection install -r requirements.yml
   ```

## Configuration

1. Create strong vault password to encrypt secrets:

   ```bash
   openssl rand -base64 128 | tr -d '\n' > .vault
   ```

2. Edit `roles/proxy/files/.env` to set your credentials and encrypt them:

   ```bash
   PROXY_LOGIN: username
   PROXY_PASSWORD: password
   ```

   ```bash
   ansible-vault encrypt roles/proxy/files/.env
   ```

3. Change VPS params in `inventory/main.yml`.
4. Customize proxy variables in `roles/proxy/vars/main.yml` if needed.
5. Customize panel variables in `roles/panel/vars/main.yml` if needed.

   > Don't forget to change **allowed_network** or you will be locked out of the VM.

## Deployment

Run the playbook:

   ```bash
   ansible-playbook playbook.yml --extra-vars "vps_ip=1.2.3.4"
   ```

## Accessing the Proxy

After deployment, the proxy will be available on SOCKS5 port 443 (TCP/UDP). You can test it with curl:

```bash
curl --socks5-hostname ${ansible_host}$:${socks_external_port} -U ${PROXY_LOGIN}:${PROXY_PASSWORD} check-host.net/ip
```

## Add the proxy to Visual Studio Code

1. Open VS Code.
2. Navigate to `File > Preferences > Settings` (or use the shortcut `Ctrl+,`).
3. In the Settings search bar, type **proxy**.
4. Locate the `Application > Proxy` section.
5. Open `setting.json` and edit proxy setting. Replace env vars with actual params.

> `http.noProxy` used for VSC hosts that use proxy but ignore proxy authorization and generates errors.  
> `http.proxyStrictSSL` can be disabled by setting it to **false** if the proxy server is not running.

```json
{
    "http.proxy": "socks5://${PROXY_LOGIN}:${PROXY_PASSWORD}@${ansible_host}$:${socks_external_port}",
    "http.proxyAuthorization": null,
    "http.proxyStrictSSL": true,
    "http.proxySupport": "fallback",
    "http.noProxy": [
        "unpkg.com",
        "avatars.githubusercontent.com",
        "default.exp-tas.com",
        "education.github.com",
        "img.shields.io",
        "embeddings.vscode-cdn.net",
        "main.vscode-cdn.net",
        "marketplace.visualstudio.com",
        "mobile.events.data.microsoft.com",
        "update.code.visualstudio.com",
        "www.gravatar.com",
        "www.vscode-unpkg.net",
        "*.gallery.vsassets.io",
        "*.gallerycdn.vsassets.io"
    ]
}
```

## 3x-UI Panel

After deployment the panel is available at:

```text
https://<ansible_host>:8443/
```

Default credentials: `admin` / `admin`

### Post-deploy configuration

1. **Change the web panel URI path** to a random UUID for security:

   ```bash
   uuidgen
   ```

   Set it in *Panel Settings → URI Path*.

2. **Change login and password** — generate a strong password:

   ```bash
   openssl rand -base64 64 | tr '+/' '-_' | tr -d '=' | head -c 64
   ```

3. **TLS certificates** are auto-generated by the playbook and mounted into the container at:

   ```text
   /root/cert/panel.crt
   /root/cert/panel.key
   ```

   Set these paths in *Panel Settings → Public Key File Path / Key File Path*.

4. **Xray routing** — in *Xray Settings*, set:
   - Direct IPs: `geoip:ru`
   - IPv4 Routing: Google (`8.8.8.8`)

### Ports

| Port | Protocol | Purpose |
|------|----------|---------|
| `8443` | TCP | 3x-UI web panel |
| `9443` | TCP | Xray inbound (configurable in panel) |
